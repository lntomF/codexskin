# CodeSkin 路线图

> English version: [ROADMAP.md](docs/en/ROADMAP.md)

本路线图是当前优先级，而不是保证交付日期。优先级会随用户反馈、目标应用更新、安全
或兼容性需求而变化。

## 核心原则

CodeSkin 应始终坚持：本地优先、可恢复、尊重隐私、不修改官方安装文件、明确兼容性
边界、安全接受社区贡献，并在视觉表达与可读性之间保持平衡。

## 当前基础能力

- Windows 桌面应用
- 本地目标应用检测
- 本地 CDP 连接与运行时视觉注入
- 背景导入与本地存储
- 主题持久化
- 主题应用验证
- 恢复官方外观流程
- Windows Release 打包

## 近期优先级

### 兼容性与可靠性

- 改进目标应用检测与失败诊断。
- 记录已测试的应用版本。
- 提高还原流程的可见性与可靠性。
- 为检测和还原增加回归覆盖。

### 更好的首次使用体验

- 清晰解释本机连接模型。
- 明确说明 CodeSkin 会修改与不会修改什么。
- 改善空状态、错误状态与重连状态。
- 让安全还原更容易找到。

### 主题质量

- 加入原创或明确可分发的默认主题。
- 改进对比度与可读性控制。
- 支持干净、低干扰和无障碍导向的预设。
- 建立安全的社区主题投稿流程。

### 社区健康

- 完善贡献指南和 Issue 模板。
- 建立兼容性报告库。
- 标注适合新人的任务。
- 发布简洁的 Release Note 和维护更新。
- 感谢文档、测试与主题贡献者。

## 后续探索

以下项目仅在获得用户验证后再考虑：按项目保存视觉配置、主题导入导出、可选设置备份、
精选社区主题库，以及录屏或演示场景的视觉模式。

## 明确不做的事

CodeSkin 不以替代 Codex、ChatGPT 或官方 Agent 工作流为目标；不修改官方安装文件；
不管理模型供应商 API Key；不分发未授权图片；不执行社区主题携带的任意脚本；也不承诺
永久兼容所有未来目标应用版本。
