# 沉浸式 CodeSkin 主题设计

**日期：** 2026-07-17  
**状态：** 已获设计批准，等待规格审阅  
**范围：** Windows 为首要验收平台；Tauri 托盘菜单保持 macOS 菜单栏兼容的实现路径。

## 1. 目标

把现有“颜色变量 + 单张背景图”的 CodeSkin 升级为沉浸式运行时主题工具，同时保持 Codex 原生交互。

最终效果：

- 侧栏、建议卡、项目选择、输入框、对话和编辑区仍是 Codex 的真实 DOM 与原生控件；CodeSkin 不用截图或覆盖式假 UI 替换它们。
- 一张用户导入的 16:9 壁纸作为连续、全窗口的底层背景。
- 首页/欢迎页使用高氛围可见度；检测到工作区、项目、对话或编辑状态时自动切换为可读性优先的专注模式。
- 导入图片后离线提取配色并生成可保存的本地主题。
- Windows 系统托盘可直接切换所有已保存主题；Tauri 的同一菜单结构兼容 macOS 菜单栏。
- 一键校验和恢复，且不修改 Codex 安装文件、`app.asar`、签名或运行参数以外的官方内容。

## 2. 非目标和不可变约束

- 不创建截图贴图、点击拦截层或模拟 Codex 控件。
- 不连接或监听非 `127.0.0.1` 的 CDP 地址；图片调色不请求外部网络。
- 不得修改官方安装包；不修改、重打包或签名 Codex 官方安装包。
- 不复用许可不明确的第三方注入脚本文本；注入逻辑维持原创实现。
- 不强制结束 Codex 进程；检测到已运行但没有 CDP 调试端口时仅提示用户正常退出。
- 未在 macOS 设备上验证前，不把 macOS 构建成功列为本轮完成条件。

## 3. 方案选择

### 3.1 不采用：整窗截图或仿制 UI

截图或重建界面无法维持 Codex 原生侧栏、项目选择、建议卡与输入框的实际交互，违反本项目目标。

### 3.2 不采用：仅 CSS 变量覆盖

仅变量覆盖不能可靠提供连续壁纸、玻璃分层、首页/工作区状态切换与主题特定遮罩。

### 3.3 采用：运行时 DOM 分层注入

通过既有的本地回环 CDP 通道，在真实 Codex 页面插入 CodeSkin 所有权明确的壁纸层、样式层和状态标记。只改变视觉属性，不替换业务 DOM，也不截获用户输入。

## 4. 注入模型

### 4.1 CodeSkin 所有权标记

每个受控页面仅由 CodeSkin 创建并清理下列标记：

- `#codeskin-wallpaper-layer`：固定定位的视觉底层；`pointer-events: none`。
- `#codeskin-runtime-style`：主题变量、背景层和组件玻璃样式。
- `data-codeskin-theme-id`：当前主题 ID。
- `data-codeskin-mode`：`ambient` 或 `focus`。
- CodeSkin 私有的观察器和新文档脚本注册 ID。

Restore 只能操作上述 CodeSkin 标记，不能删除或重写 Codex 自己创建的节点和样式。

### 4.2 连续壁纸与安全区

壁纸层定位在页面最底层，使用 `background-size: cover`、`background-position: center`，覆盖整个视口。它永远不接收点击、键盘或拖放事件。

样式层根据当前模式为真实 Codex 元素增加半透明背景、边框、阴影及可读性遮罩。所有规则使用 CodeSkin CSS 变量；不重建侧栏、卡片或输入框。

### 4.3 组件样式范围

优先应用到具有稳定语义的导航、侧栏、主内容、卡片、列表行、表单输入、可编辑区域、代码块及对话容器。选择器必须带 CodeSkin 根标记，避免页面未激活时产生任何影响。

未知或未匹配的组件保留 Codex 官方外观。不能为了“视觉完整”而使用广泛的 `*` 覆盖或强制改变布局。

## 5. 自动页面状态

### 5.1 状态

- `ambient`：欢迎页、首页或没有正在进行任务的页面。壁纸高可见度，卡片和导航是轻至中等强度玻璃层。
- `focus`：项目、会话、编辑器、任务或输入工作区。壁纸仍连续铺在底层，但主内容和输入区获得更强遮罩与更高文字对比度。

### 5.2 判定机制

注入脚本使用 `MutationObserver` 和一组 DOM 语义信号计算模式。信号包括工作区容器、编辑器/可编辑输入、对话记录、项目导航以及欢迎/建议区域的存在情况。

每次明显 DOM 变化后重新计算，并通过 `data-codeskin-mode` 驱动 CSS。若信号冲突、页面初始加载未完成或 Codex 更新导致选择器失效，默认 `focus`，优先保证输入和阅读可用性。

### 5.3 可验证性

Verify 要读取主题标记、壁纸节点、样式节点和当前模式，并对每个 CDP target 返回明确状态。不能仅报告“CDP 请求成功”。

## 6. 自动主题生成

### 6.1 输入

支持 PNG、JPEG、WebP。沿用现有图片大小和尺寸限制；图片仅写入 CodeSkin 本地应用数据目录。

### 6.2 本地调色

用现有 `image` crate 缩放图像后进行本地像素采样。无需引入第三方调色库或联网能力。

算法输出：

- 图片总体明暗倾向；
- 主色、次色和可用强调色；
- 适配明暗倾向的前景文字与弱文字颜色；
- `ambient` / `focus` 模式的主内容、侧栏和卡片遮罩强度；
- 边框高光与输入区对比色。

算法优先稳定和可读性：强调色若与文字或遮罩对比度不足，则调整其亮度/饱和度。导入后用户无需手动配置颜色即可生成主题。

### 6.3 生成主题元数据

用户主题至少包含：

- 唯一 ID；
- 显示名称（默认为图片文件名，GUI 可改名）；
- 壁纸本地路径；
- 推导后的颜色；
- `ambient` 与 `focus` 遮罩参数；
- 创建和更新时间；
- 主题来源（内置或用户导入）。

## 7. 存储和托盘

### 7.1 持久化

将现有“已选主题 + 单个背景路径”扩展为本地 `themes.json`：包含内置主题引用、用户主题列表、当前主题 ID。用户图片保存到 CodeSkin 的应用数据目录；JSON 不保存原始二进制。

存储迁移必须兼容旧设置：旧的当前背景图和选中内置主题，在首次加载时转换为对应的用户主题或保留为无壁纸内置选择，不能让现有用户配置丢失。

### 7.2 系统托盘

菜单结构：

1. 显示 CodeSkin；
2. `主题` 子菜单，列出所有已保存主题，当前项显示选中状态；
3. 恢复 Codex 原始外观；
4. 退出。

选择主题后立即保存当前 ID 并向已连接 target 注入。没有可连接 Codex target 时仅保存选择，之后连接成功时应用。

Windows 是本轮实际验证对象；Tauri tray 菜单使用跨平台 API，macOS 映射到菜单栏状态菜单。

## 8. GUI

GUI 包含：

- 内置主题和用户壁纸主题的统一主题网格；
- 壁纸缩略图与派生调色预览，不使用伪 Codex 页面截图；
- “导入壁纸并生成主题”流程；
- 新主题名称编辑和保存；
- 当前主题的明暗判断与模式遮罩摘要；
- 既有的连接、应用、Verify、Restore 操作；
- Verify 详细 target 结果：主题 ID、壁纸层、样式层、当前模式、失败原因。

## 9. Restore、重连与失败行为

Restore 删除 CodeSkin 的壁纸层、样式层、`data-codeskin-*` 属性、观察器注册和新文档脚本；它不修改官方文件，也不尝试恢复未由 CodeSkin 改动的官方样式。

CDP 断线重连后，现有连接管理器重建 target 注册，并根据当前已保存主题恢复注入。刷新/新窗口时保留原有“新文档注入 + 页面加载事件 fallback”策略。

若页面结构更新：

- 主题的基础层和壁纸层仍可注入；
- 组件定向样式或状态识别可能降级；
- Verify 显示降级详情；
- 页面默认使用 focus 遮罩；
- 用户可 Restore 恢复官方外观。

## 10. 实施顺序和验收

1. 扩展主题模型和本地存储，保留旧设置迁移。
2. 本地图片调色与主题生成，覆盖明暗、颜色和参数单元测试。
3. 重写原创注入脚本：壁纸层、模式观察、玻璃样式和 Verify 字段。
4. 扩展 Rust 注入 payload、AppState、命令和重连生命周期。
5. 改造 GUI：导入生成、主题保存、卡片预览、验证详情。
6. 动态构建托盘主题子菜单，并接入立即应用。
7. 分模块执行 `cargo test` / `cargo check`；最终执行 `npm.cmd run build:desktop`。
8. 在真实 Codex 中验证：导入图片、自动主题、真实控件交互、ambient/focus 切换、刷新保持、托盘切换、Verify、Restore、Codex 重启后的重连。

## 11. 验收标准

- `cargo build --release` 和 `npm.cmd run build:desktop` 成功。
- 一张 16:9 本地图片能生成并保存可复用主题。
- 壁纸连续铺满 Codex 窗口底层，且不遮挡或接管原生控件。
- 首页自动显示 ambient；进入任务、项目、对话或编辑工作区后自动显示 focus。
- 输入框、项目选择、侧栏、建议卡与主内容仍可实际交互。
- 托盘列出并能直接切换所有保存的主题。
- Verify 确认主题、壁纸层、样式层和模式；Restore 清理所有 CodeSkin 标记。
- CDP 连接严格只到 `127.0.0.1`，不改官方安装包。
