# CodeSkin Asset Policy

> 中文版：[查看素材政策](../../ASSET_POLICY.md)

This policy defines which visual assets may be included in the public CodeSkin
repository, documentation, theme packs, and release builds.

## Allowed assets

You may submit an asset only when at least one of the following is true:

1. You created it yourself and own the right to redistribute it.
2. You have written permission from the copyright holder.
3. It is published under a license that clearly permits redistribution in this
   project.
4. It is a public-domain or CC0 asset and you retain its source information.
5. It is an abstract visual created entirely from CSS, gradients, shapes, or
   other project-authored visual primitives.

## Assets that must not be submitted

Do not submit the following to this repository or to an official CodeSkin
release:

- assets containing personal information, private repositories, API keys,
  account names, chat content, or confidential project details.

## Theme submission requirements

Every visual-theme submission must include a theme name, author attribution,
preview image, source or original-work statement, and a clear license or
permission statement. The maintainer may reject or remove any asset when its
ownership, license, safety, quality, or relevance is unclear.

## Local-only personal assets

CodeSkin users may import their own local wallpapers for personal use. Personal
local assets must not be committed to this public repository, included in
sample theme packs, or uploaded as official project releases unless their
redistribution rights are verified.

## Reporting a concern

If you believe an included asset violates your rights, please report it through
the repository's private security-reporting channel. Include the file path,
reason for the report, and any proof of ownership or licensing information.
