# Getting Help with CodeSkin

> 中文版：[查看支持指南](../../SUPPORT.md)

## Before asking for help

Check the latest release notes, existing GitHub Issues, the troubleshooting
section in `README.md`, and whether your Codex / ChatGPT Desktop version is
listed as compatible. Reproduce the issue after restarting CodeSkin and the
target application normally when possible.

## Where to ask

### Bug or compatibility problem

Open an Issue using the relevant template when installation fails, CodeSkin
cannot detect or connect to the target application, a theme does not apply,
restoration does not work, or a new target-app version changes behavior.

### Feature idea

Open a feature request when you can describe the user problem, the current
workaround, who would benefit, and why the change matters beyond a cosmetic
preference.

### Theme submission

Use the theme-submission template only for assets that meet
[ASSET_POLICY.md](ASSET_POLICY.md).

### Security issue

Do not open a public Issue. Read [SECURITY.md](SECURITY.md) and use private
vulnerability reporting.

## What makes a helpful request

Include CodeSkin, Windows, and target-app versions; a short reproducible
sequence; expected and actual behavior; and sanitized screenshots or logs. Do
not include API keys, tokens, private repository paths, customer information,
private task or chat content, or unlicensed third-party images.

## Maintainer scope

CodeSkin is an independent, unofficial project. Support is best-effort and
focuses on reproducible issues in supported versions of the application.
